# Lambda Layer ライフサイクル (Lambda Layer Lifecycle)

## ビルドのインプット情報 (Build Inputs)

コンテンツの同一性（コンテンツハッシュ）は、Gemfile.lockの内容、選択されたBundlerグループ、ランタイム、CPUアーキテクチャ、ビルドイメージの識別情報、包含/除外ルール、ソースディレクトリ、およびVeltrunodeのビルドスキーマバージョンによって識別されます。

## レイヤーの再利用 (Reuse)

新しいLayerバージョンを発行（Publish）する前に、Veltrunodeはコンテンツハッシュ値をマニフェストおよびオプションのLayer説明メタデータと比較します。コンテンツに変更がない場合は、既存のLayerバージョンを再利用します。バージョンが存在しないか検証できない場合は、不確実な推測を避け、新しいバージョンを発行します。

## ネイティブ拡張機能 (Native Extensions)

ネイティブ拡張を含むGemは、CPUアーキテクチャに一致する固定（ピン留め）されたAmazon Linux互換コンテナ内でビルドされます。クロスアーキテクチャのビルドには、互換性のあるコンテナランタイムが必要です。ビルドログには、使用したコンテナイメージのダイジェスト（Digest）が記録されます。

## 保持ポリシーとクリーンアップ (Retention)

古いバージョンのクリーンアップ（Prune）はオプトイン方式です。他のリソースから現在参照されているバージョンが削除されることはありません。ドライラン（`dry-run`）を実行すると、削除候補のバージョンと現在参照されている情報が一覧表示されます。本番環境でのクリーンアップ実行には、明示的な確認が必要です。

## サイズの診断 (Size Diagnostics)

ビルド時に、解凍時および圧縮時のファイルサイズ、最も大きなエントリ、Layer間や関数パッケージ間での重複ファイル、および推奨される配置場所がレポートされます。ただし、初期フェーズでは、関数、Layer、EFSの間での依存関係の自動移動機能はサポートされません。
