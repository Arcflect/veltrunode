# セキュリティポリシー (Security Policy)

## サポートされているバージョン

現在、本プロジェクトは開発・設計段階です。セキュリティアップデートは最新のリリース（メインブランチの最新コード）に対してのみ提供されます。

| バージョン | サポート状況 |
| ---------- | ------------ |
| 最新版     | :white_check_mark: |
| 旧バージョン | :x:          |

## 脆弱性の報告方法

資格情報の漏洩、任意コードの実行、パッケージの改ざん、IAM権限昇格、または重大なデプロイの欠陥など、セキュリティに関する脆弱性については、パブリックなIssueを開かないでください。

脆弱性を発見した場合は、以下の手順でプライベートに報告してください。

1. GitHubのリポジトリに用意されている「プライベート脆弱性報告（Private vulnerability reporting）」機能を使用するか、メンテナー宛てに直接ご連絡ください。
2. 報告の際は、以下の情報を含めてください。
   - 影響を受けるコミット/バージョン
   - 再現手順
   - 影響度
   - 推奨される修正案
3. **重要**: アクティブなAWSの認証情報（アクセスキー等）は絶対に報告内容に含めないでください。

報告を受領後、調査と修正対応を進め、必要に応じてセキュリティアドバイザリを公開します。
